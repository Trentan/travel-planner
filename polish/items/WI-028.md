## Problem

On mobile, the combination of the sticky menu bar, tab navigation, and city navigation consumes nearly 30-40% of the vertical viewport. This leaves very little room for viewing itinerary cards or budget tables.

## Before (current state)

> Screenshot: `../screenshots/before/mde-01-home.png`
> Mode: MDE
> Callout: Notice how little content is visible below the navigation stack.

## Proposed fix

Implement a "condensed chrome" mode that triggers when the user scrolls down. Hide the menu bar and/or combine tab and city nav into a single compact row. Show again when scrolling up.

**Mockup Creation:**
If the change is visual, you MUST create a visual mockup (Option A or B in Step 6) and link it here.

## After (proposed state description)

As the user scrolls down, the navigation stack collapses to a minimal height (e.g. 50px), maximizing content space.

## Acceptance criteria

- [ ] Navigation collapses on scroll down.
- [ ] Navigation restores on scroll up.
- [ ] No "jank" or layout shifts during transition.
- [ ] Regression check passes: `node scripts/regression-city-nav.js`

## How to implement

1. Add a scroll listener in `js/ui.js`.
2. Toggle a class on `body` or `html`.
3. Add CSS transitions for the nav elements.
