## Problem

When a suggested activity is moved onto a day, the day item copies only the activity title. The suggested activity's category emoji is shown in the idea list, but it is lost from the planned day activity.

## Before

Screenshot: [WI-029-after-4.png](../screenshots/after/WI-029-after-4.png)

Mode: MDE

Callout: Assigned activities keep their text, cost, and duration, but the category emoji is not carried into the day item.

## Proposed Fix

Build the assigned day item text from the suggested activity category emoji plus the original title. Make reassignment and removal matching tolerant of both old no-emoji activity text and new emoji-prefixed activity text.

Proposal: [WI-029-proposal.png](../screenshots/after/WI-029-proposal.png)

## After

Latest review screenshot: [WI-034-after.png](../screenshots/after/WI-034-after.png)

Assigned activities now retain the category emoji as the first part of the planned day item, including both drag/drop assignment and the touch-friendly Assign picker.

## Acceptance Criteria

- [x] Assigning a suggested activity to a day prefixes the day item with the activity category emoji.
- [x] Moving an assigned activity removes the previous day item and keeps the emoji on the new day.
- [x] Clearing an assigned activity removes the emoji-prefixed day item.
- [x] Existing no-emoji assigned items can still be matched and removed.

## How Implemented

1. Added a shared assignment text helper that prefixes assigned day items with `getActivityEmoji(activity.category)`.
2. Updated assignment, reassignment, clearing, deletion, and edit synchronization to match both legacy title-only items and new emoji-prefixed items.
3. Extended the activity assignment smoke tests to assert the category emoji prefix for drag/drop and touch assignment paths.
