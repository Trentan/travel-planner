## Problem

The transport tab rendered as a dense card-like mobile layout. The table columns were not clearly separated, and the header did not stay visible while scrolling.

## Review Notes

- Keep the transport view table-like on mobile.
- Restore clear column headings in the mobile header.
- Keep carrier/provider details visible, along with journey hours.
- Make the edit action look functional and keep it from overlapping the last column.
- Keep the mobile view simple; hide the unreadable segment subtable and leave the parent journey row as the readable summary.
- Leave delete in the journey dialog instead of the row action column.
- Show the compact mobile column headings.
- Keep the multi-leg toggle visible and put the type emoji before the duration line.
- Put the status control at the top of the carrier column to keep it clear of the provider and cost text.
- Mobile detailed view should match compact visually on mobile.
- Keep the table headings visible in mobile detailed mode too.
- Change the row action column label to `Edit` and use a tap-friendly pencil chip.
- Keep the journey legs readable in the compact mobile treatment.
- Use the same standard blue pencil edit icon as desktop mode.
- Make the journey description itself the expand/collapse trigger so the row uses less space.
- Keep the collapsible journey title visually aligned with the other row text; only the `via` fragment should be italic.
- Make the collapsible journey title read like the other row text in mobile, not a bigger or heavier heading.

## Proposed fix

Keep the transport view as a table on mobile, but simplify it to four clear columns: Journey, Schedule, Status/Cost, and Actions. Add a sticky mobile header row so the column labels remain visible while scrolling.
