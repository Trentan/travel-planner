## Problem

Suggested activities rely on HTML5 drag-and-drop, which does not work on most mobile touch browsers. Mobile users need a direct way to assign ideas to specific days.

## Before

Screenshot: [mde-05-itinerary.png](../screenshots/before/mde-05-itinerary.png)

Mode: MDE

Callout: The drag handle is visible but unresponsive to touch.

## Proposed Fix

Improve the Assign button workflow for mobile so tapping Assign opens a touch-friendly day picker with clear targets and current-assignment context.

Proposal: [WI-029-proposal.png](../screenshots/after/WI-029-proposal.png)

## After

Latest review screenshot: [WI-029-after-4.png](../screenshots/after/WI-029-after-4.png)

In MDE and MCO, tapping Assign opens a full-height mobile picker. Each destination day is a large tap target, and successful assignment shows visible feedback.

## Acceptance Criteria

- [x] Activities can be assigned to days on mobile without a mouse.
- [x] Visual feedback appears when an activity is successfully assigned.
- [x] Tap targets remain >=44px.
- [x] Regression check passes via available script: `node tests/city-nav-regression.js`

## How Implemented

1. Reworked the Assign modal into a dedicated touch picker with larger day rows, route labels, current-state styling, and escaped display text.
2. Kept the Assign/Move action visually compact while maintaining a 44px mobile tap target.
3. Added reusable assignment feedback for modal assignment and drag/drop assignment.
