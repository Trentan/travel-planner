## Problem

Suggested activities rely on HTML5 drag-and-drop, which doesn't work on most mobile touch browsers. Mobile users have no way to assign "Ideas" to specific days other than the "Assign" button which opens a modal.

## Before (current state)

> Screenshot: `../screenshots/before/mde-05-itinerary.png`
> Mode: MDE
> Callout: The "⠿" drag handle is visible but unresponsive to touch.

## Proposed fix

Improve the "Assign" button workflow for mobile. Instead of relying on drag, allow tapping "Assign" to enter a "placement mode" where days become targetable, or improve the modal selection.

**Mockup Creation:**
If the change is visual, you MUST create a visual mockup (Option A or B in Step 6) and link it here.

## After (proposed state description)

In MDE and MCO, tapping the "Assign" button on an activity allows the user to easily pick a destination day without complex dragging.

## Acceptance criteria

- [ ] Activities can be assigned to days on mobile without a mouse.
- [ ] Visual feedback when an activity is successfully assigned.
- [ ] Tap targets remain ≥44px.
- [ ] Regression check passes: `node scripts/regression-city-nav.js`

## How to implement

1. Update `js/dragdrop.js` to support touch events or add a tap-to-select flow.
2. Ensure the "Assign" button in `js/itinerary.js` is prominent in mobile modes.
