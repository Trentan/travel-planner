## Problem

The current map is a static SVG approximation with a hardcoded list of city coordinates. Many cities are unmapped, and there is no interactivity (zoom, pan, real terrain). This undermines the "Plan" aspect of the app. Post-advances audit confirms this is the primary functionality gap.

## Before (current state)

> Screenshot: `../screenshots/before/dde-09-map.png`
> Mode: DDE
> Callout: The map is a simple SVG with continent outlines and curved lines between dots.

## Proposed fix

Replace the SVG implementation in `js/map.js` with Leaflet.js. Use OpenStreetMap tiles. Map trip legs as polylines and cities as markers.

**Mockup Creation:**
If the change is visual, you MUST create a visual mockup (Option A or B in Step 6) and link it here.

## After (proposed state description)

A fully interactive map with zoom and pan. Trip legs are connected by real geographic lines (or great circles). Clicking a marker shows city details. All modes show a responsive map container.

## Acceptance criteria

- [ ] Leaflet.js integrated and rendering real tiles.
- [ ] Markers for all cities in the trip.
- [ ] Polylines connecting trip legs.
- [ ] Zoom/pan functional in all modes.
- [ ] Regression check passes: `node scripts/regression-city-nav.js`

## How to implement

1. Add Leaflet CSS/JS to `index.html`.
2. Refactor `js/map.js` to initialize a Leaflet map in `#journey-map-view`.
3. Use `citiesData` to place markers.
4. Use `appData` and `journeys` to draw paths.
