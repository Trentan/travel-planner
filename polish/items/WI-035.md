# WI-035: Map Geocoding & Marker Refinement

## Status
- **Priority**: High
- **Status**: In Progress
- **Assigned**: Gemini CLI


## Goal
Improve the journey map's city mapping reliability and simplify visual markers.

## Requirements
1. **Online Geocoding Fallback**: Implement a search function that queries an online API (like Nominatim) if a city is not found in the local database.
2. **IATA & Metadata Enrichment**: Use geocoding results to try and find/confirm IATA codes.
3. **Simplified Markers**: Markers should only display the first sequence number of a visit to reduce visual clutter. 
4. **Directional Flow**: Ensure polylines clearly demonstrate the back-and-forth travel path.
5. **Offline-First**: Ensure online-found coordinates are saved locally to maintain offline support.

## Tasks
- [x] Implement `searchCityOnline` utility in `js/data.js`.
- [x] Add "Search Online" button to Manage Cities dialog.
- [x] Refactor `js/map.js` to simplify marker numbering (show only first visit).
- [x] Update `js/map.js` marker popups to show full visit history (e.g., "First visit: 1, also visited at: 5").
- [x] Validate routing with return legs.

## Technical Notes
- API: Nominatim (OpenStreetMap) for free, no-key geocoding (respecting rate limits).
- Storage: Coordinates fetched online must be saved to `citiesData`.
