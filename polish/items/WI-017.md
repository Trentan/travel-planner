## Problem

The AI Builder modal opens with blank fields even if a trip is already loaded. Users have to re-type their trip title, dates, and cities to generate a prompt.

## Before

Screenshot: [dde-11-ai-builder.png](../screenshots/before/dde-11-ai-builder.png)

Mode: DDE

Callout: "Trip Title", "Dates", and "Cities" fields are empty.

## Proposed Fix

Update `openAIDialog` to gather data from `appData` and `titleData` and pre-fill the input fields if they are currently empty.

Proposal: [WI-017-proposal.png](../screenshots/after/WI-017-proposal.png)

## After

Latest review screenshot: [WI-017-after.png](../screenshots/after/WI-017-after.png)

When opening AI Builder, the "Trip Title" shows the current trip name, "Dates" shows the trip range plus known flights, and "Cities" lists the destination cities.

## How Implemented

1. Added AI Builder pre-fill helpers that read the current title, itinerary date range, journey flights, and destination cities.
2. Wired `openAIDialog` to pre-fill only empty fields so user edits are preserved.
