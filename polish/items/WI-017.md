## Problem

The AI Builder modal opens with blank fields even if a trip is already loaded. Users have to re-type their trip title, dates, and cities to generate a prompt.

## Before (current state)

> Screenshot: `../screenshots/before/dde-11-ai-builder.png`
> Mode: DDE
> Callout: "Trip Title", "Dates", and "Cities" fields are empty.

## Proposed fix

Update `openAIDialog` to gather data from `appData` and `titleData` and pre-fill the input fields if they are currently empty.

**Mockup Creation:**
If the change is visual, you MUST create a visual mockup (Option A or B in Step 6) and link it here.

## After (proposed state description)

When opening AI Builder, the "Trip Title" shows the current trip name, "Dates" shows the trip range, and "Cities" lists all destinations.

## Acceptance criteria

- [ ] Fields pre-filled on modal open.
- [ ] User can still edit the pre-filled text.
- [ ] Works in all four modes.
- [ ] Regression check passes: `node scripts/regression-city-nav.js`

## How to implement

1. In `js/ui.js`, modify `openAIDialog` to call a pre-fill helper.
2. Implement pre-fill logic using `appData` and `meta` data.
