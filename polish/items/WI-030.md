## Problem

The mobile menu sheet is a dense grid of equally-sized buttons. It's hard to distinguish between configuration, functional actions, and data actions.

## Before (current state)

> Screenshot: `../screenshots/before/mde-04-menu-open.png`
> Mode: MDE
> Callout: The grid layout makes every action look the same.

## Proposed fix

Reorganize the mobile menu into sections with small headers. Use different button styles for primary vs secondary actions.

**Mockup Creation:**
If the change is visual, you MUST create a visual mockup (Option A or B in Step 6) and link it here.

## After (proposed state description)

The mobile menu has clear groupings like "Trip Settings", "Planning Tools", and "Export & Files".

## Acceptance criteria

- [ ] Menu organized into logical sections.
- [ ] Section headers added.
- [ ] Regression check passes: `node scripts/regression-city-nav.js`

## How to implement

1. Update `index.html` mobile menu sheet structure.
2. Add CSS for mobile menu section headers.
