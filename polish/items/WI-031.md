## Problem

Desktop Compact mode uses a legacy list-based renderer, while Mobile Compact mode uses a modern swipe-card pager. This creates a disjointed experience.

## Before

Screenshot: [dco-04-itinerary.png](../screenshots/before/dco-04-itinerary.png)

Mode: DCO

Callout: Compare the desktop list style to the mobile card style.

## Proposed Fix

Refactor `js/itinerary.js` to use a single Compact visual system. Modernize the legacy list to match the card aesthetics.

Proposal: [WI-031-proposal.png](../screenshots/after/WI-031-proposal.png)

## After

Latest review screenshot: [WI-031-after.png](../screenshots/after/WI-031-after.png)

Compact mode now uses the same card/pager renderer across desktop and mobile instead of switching desktop to the legacy list.

## How Implemented

1. Routed compact itinerary rendering through the modern compact card/pager renderer for both desktop and mobile.
2. Added desktop-safe compact card, chip, rail, and pager styles so the shared compact renderer is fully styled outside the mobile breakpoint.
