## Problem

Desktop Compact mode uses a legacy list-based renderer, while Mobile Compact mode uses a modern swipe-card pager. This creates a disjointed experience.

## Before (current state)

> Screenshot: `../screenshots/before/dco-04-itinerary.png`
> Mode: DCO
> Callout: Compare the desktop list style to the mobile card style.

## Proposed fix

Refactor `js/itinerary.js` to use a single "Compact" visual system. Modernize the legacy list to match the card aesthetics.

**Mockup Creation:**
If the change is visual, you MUST create a visual mockup (Option A or B in Step 6) and link it here.

## After (proposed state description)

Compact mode feels like a single coherent "view" of the app, regardless of viewport size.

- [ ] Regression check passes: `node scripts/regression-city-nav.js`
