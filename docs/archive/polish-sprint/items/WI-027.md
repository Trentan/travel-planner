## Problem

Transport rows remained significantly taller (~68px) than accommodation rows, causing an inconsistent visual rhythm and reducing information density on desktop and compact views.

## Proposed fix

1. Drastically reduce padding on all transport table cells.
2. Remove `min-height` constraints on journey name containers.
3. Tighten `line-height` for multi-line text.
4. Add forced overrides to ensure consistency across the fragmented CSS.

## Implementation details

Appended forced overrides to `style.css`:
- `padding: 0.35rem 0.4rem !important` for transport cells.
- `min-height: 0 !important` for journey name main.
- `line-height: 1.2 !important` for text.
- Even tighter padding (`0.2rem`) for compact mode.

## Acceptance criteria

- [ ] Transport row height is comparable to accommodation row height (~40px).
- [ ] No content is clipped or unreadable.
- [ ] Compact mode is even denser.
- [ ] Regression check passes.
