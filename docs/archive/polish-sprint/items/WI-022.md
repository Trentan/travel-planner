## Problem

The packing tab mixes guides and checklist content without a strong progress summary. Users cannot immediately see how much remains to pack.

## Before (current state)

## Before image

[WI-022 before](../screenshots/before/desktop-08-packing.png)

## Proposed fix

Add a compact progress strip per packing area and visually separate guide content from actionable checklists.

## Proposal image

[WI-022 proposal](./proposals/WI-022-proposal.png)

## After (proposed state description)

Packing starts with progress and next actions, then detailed lists below. Progress bars are consolidated into the guide panel toolbar as interactive navigation links.

## Acceptance criteria

- [x] Packing tab has an at-a-glance progress indicator (consolidated toolbar).
- [x] Guide panels are visually secondary to checklist tasks.
- [x] Progress bars are minimal and color-coded.
- [x] Progress bars act as navigation links to jump to packing areas.
- [x] Vertical spacing is optimized to remove large gaps.
- [x] Regression check passes: `node tests/city-nav-regression.js`
