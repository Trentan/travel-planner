## Problem

Large portions of the UI are styled inline in HTML and JS builders. This makes responsive polish harder and causes repeated one-off spacing, colors, and typography.

## Proposed fix

Move repeated inline styles into named classes and CSS variables, starting with modal forms, tab headers, and itinerary detail blocks.

```css
/* BEFORE */
.target-selector { /* current layout clips, wraps, or undersizes at the tested viewport */ }

/* AFTER */
.target-selector { /* responsive layout meets the acceptance criteria for WI-024 */ }
```
