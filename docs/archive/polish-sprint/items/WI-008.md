## Problem

`#mainTitle` and `#mainSubtitle` are contenteditable by default in edit mode. On mobile, casual taps in the hero can focus the fields and open the keyboard even when the user is trying to scroll.

## Proposed fix

Disable direct contenteditable on mobile unless an explicit Edit Header action is selected, or require read-only mode by default on mobile.
