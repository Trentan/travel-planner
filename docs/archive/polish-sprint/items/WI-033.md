## Problem

The WI-031 compact unification made desktop compact mode feel too much like the mobile itinerary. Desktop lost the quick stacked overview that makes it useful for scanning a whole trip.

## Before

Screenshot: [WI-031-after.png](../screenshots/after/WI-031-after.png)

Mode: DCO

Callout: Desktop compact is constrained by the mobile city-swipe pager instead of showing the itinerary as a scannable desktop stack.

## Proposed Fix

Keep the shared compact card styling, but split the desktop compact itinerary into a stacked desktop renderer. Mobile compact should continue using the swipe city pager.

Proposal: [WI-031-proposal.png](../screenshots/after/WI-031-proposal.png)

## After

Latest review screenshot: [WI-033-after.png](../screenshots/after/WI-033-after.png)

Desktop compact mode now renders all city cards in a stacked itinerary with the food quest panel and day pager side by side where space allows. Mobile compact still uses the city swipe pager.

## Acceptance Criteria

- [x] Desktop compact itinerary is scannable without the city swipe shell.
- [x] Mobile compact itinerary keeps the swipe city pager.
- [x] Regression check passes via available script: `node tests/city-nav-regression.js`

## Review Notes

- 2026-05-18: User feedback from WI-031: desktop compact itinerary needs a rework; the mobile-style compact pager is not right for desktop.

## How Implemented

1. Added a desktop-only compact itinerary renderer that stacks compact leg cards.
2. Kept the existing mobile compact city-swipe renderer for mobile viewports.
3. Added desktop compact layout styles so each card remains compact while exposing more of the itinerary at once.
